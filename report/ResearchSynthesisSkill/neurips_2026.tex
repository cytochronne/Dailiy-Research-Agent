\begin{abstract}
This outline is structured for the required three-page NeurIPS-format individual Skill report. It focuses on \texttt{ResearchSynthesisSkill}, the synthesis-side Skill that turns ranked arXiv recommendations into structured daily briefings and selected-paper explanations while preserving explicit evidence boundaries. The final report should explicitly cover functionality, references, results, analysis, and visualization.
\end{abstract}

\section{Functionality}

\paragraph{Skill purpose.}
Define \texttt{ResearchSynthesisSkill} as the Skill that explains and summarizes the papers selected by the recommendation-side Skill. It answers: what does each recommended paper contribute, why should the user read it, how do the Top-K papers compare, and what evidence supports each generated claim?

\paragraph{Inputs.}
List the inputs consumed from the Agent workflow:
\begin{itemize}
  \item Top-K \texttt{Recommendation} rows with rank, score, paper metadata, evidence source, and ranking rationale.
  \item Candidate-pool context, \texttt{QueryPlan}, \texttt{RetrievalQuery}, retrieval metadata, and ranking metadata.
  \item Optional selected paper, explanation mode, cached full text, or user-provided full text for deep explanation.
\end{itemize}

\paragraph{Outputs.}
Describe the outputs: \texttt{PaperBriefingItem}, \texttt{DailyBriefing}, \texttt{PaperDeepExplanation}, evidence-bound claims, reading priorities, Top-K comparisons, trend overview, and a structured evidence boundary. State explicitly that the default daily briefing does not claim PDF or full-text evidence unless full text is actually available for selected-paper explanation.

\section{Method and implementation}

\paragraph{Per-paper extraction.}
Explain that \texttt{extract\_paper(...)} converts each recommendation into a structured \texttt{PaperBriefingItem}: summary, contributions, methods, relevance rationale, problem, approach, reading guide, and evidence-bound claims. Provider failures produce deterministic fallback items rather than dropping the paper.

\paragraph{Daily briefing generation.}
Explain that \texttt{generate\_briefing(...)} combines extraction results, candidate-pool context, query-plan context, and ranking metadata into an enhanced \texttt{DailyBriefing}. The briefing contains an executive summary, summary table, highlighted paper, paper-level items, trend overview, Top-K comparisons, reading priorities, and evidence boundary.

\paragraph{Selected-paper deep explanation.}
Explain that \texttt{explain\_paper(...)} supports \texttt{method}, \texttt{experiment}, and \texttt{limitations} modes. It prefers full text when supplied or cached; otherwise it falls back to abstract, then metadata. Each fallback is explicitly labeled so the user can distinguish full-paper analysis from abstract-only or metadata-only output.

\section{References and related work}

Use this section to position the Skill relative to evidence-grounded summarization:
\begin{itemize}
  \item Retrieval-augmented generation motivates separating retrieval evidence from generated synthesis \citep{lewis2020}.
  \item Faithfulness work in summarization motivates evidence labels and forbidden full-text claims \citep{maynez2020}.
  \item Hallucination surveys motivate explicit abstentions and source boundaries when evidence is missing \citep{ji2023}.
  \item arXiv metadata and abstracts define the default evidence available to the briefing \citep{arxivapi}.
\end{itemize}

\section{Analysis}

\paragraph{What works and why.}
Argue that the Skill improves usefulness because it turns Top-K rows into reading decisions. Evidence labels reduce hallucination risk, Top-K comparisons make recommendations easier to compare, reading priorities help users decide what to read first, and mode-specific explanations expose missing information instead of hiding it inside a generic paragraph.

\paragraph{Ablations and error analysis.}
Include these analyses in the final report:
\begin{itemize}
  \item Enhanced briefing versus generic summary: the evaluator fails generic text even when sections exist.
  \item Default briefing with and without forbidden full-text claims: PDF/full-text claims are rejected unless full-text evidence was used.
  \item Full-text explanation versus abstract/metadata fallback.
  \item Explanation completeness by mode, with the 0.86 method-mode result showing one intentionally missing required field.
\end{itemize}

\paragraph{Limitations.}
State the limits plainly: default daily briefings are based on metadata, abstracts, ranking evidence, and candidate-pool context; full-text analysis is only for selected-paper explanation; fake-provider results test structure rather than human preference; and current explanation completeness does not measure factual correctness beyond available evidence labels.
